Introduction

StudScore API | One of the best API for student data migration.

This documentation aims to provide all the information you need to work with our API.

As you scroll, you'll see code examples for working with the API in different programming languages in the dark area to the right (or as part of the content on mobile). You can switch the language used with the tabs at the top right (or from the nav menu at the top left on mobile).

Base URL

https://api.test

Authenticating requests

This API is authenticated by sending an Authorization header with the value "Bearer {YOUR_AUTH_KEY}".

All authenticated endpoints are marked with a requires authentication badge in the documentation below.

You can retrieve your token by visiting your dashboard and clicking Generate API token.

Endpoints

Get a JWT via given credentials.

Example request:
curl --request POST \
    "https://api.test/api/v1/login" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"email\": \"jeremie65@example.com\",
    \"password\": \"f\"
}"

const url = new URL(
    "https://api.test/api/v1/login"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "email": "jeremie65@example.com",
    "password": "f"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Users

API endpoints for managing app Users. Actions on this resource is only allow to 'Admin' and 'Client' users.

Display or send a listing of the Users resource.

requires authentication

https://api.test/api/v1/users

Allow autorised user to get listing of Users resource.

Get linsting of Users resource in specifique format:

https://api.test/api/v1/users?format=value

format parmeter can have as value json or csv.

Additional functionnalities:

With this endpoint, you can chain multiple parameters on URI to make filter on Users resource retrive from database. So If you want the User resources which contain name like luc you have to use the parameters name:

Ex: https://api.test/api/v1/users?name=luc

Note: This filter is only allow for [name, type_user, email, valid] fields
